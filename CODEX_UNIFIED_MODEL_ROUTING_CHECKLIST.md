# TokenPilot 统一模型调用与无感路由改造 Checklist

> 用途：作为本次改造的唯一推进清单。完成一项即把 `[ ]` 改为 `[x]`，并在同一项后附测试命令、接口结果或页面截图位置；不要再新增零散进度文档。

## 目标结果

- 应用只配置并调用一个稳定的“虚拟模型”名称，不感知最终使用 LiteLLM、OpenAI、Anthropic 或其他兼容服务。
- TokenPilot 后台统一维护“调用连接、真实模型、虚拟模型、路由条件、降级顺序、AIU 单价和用户额度”。
- LiteLLM 是一种调用连接，不再拥有独立的数据模型、路由规则或统计链路。
- Node.js 和 Python SDK 能拉取已发布配置，在应用进程内完成选路、额度预占、调用、降级、结算和用量上报。
- 所有调用方式进入同一条事件、AI Cost、AIU、用户和 ClickHouse 报表链路。
- 供应商密钥和请求内容留在应用侧；TokenPilot 不保存密钥、提示词、回复正文或工具参数。
- 已注册连接之间的模型切换只需发布配置，不要求应用重新部署。

## 已确定的产品与技术决策

- [x] 唯一调用关系为：`应用 -> 虚拟模型 -> 真实模型 -> 调用连接 -> 模型服务`。
- [x] “抽象模型”统一更名为“虚拟模型”，代码、接口、页面和文档不再出现旧名称。
- [x] 真实模型代表可被实际请求、计价并换算 AIU 的模型实例。
- [x] 调用连接代表如何调用模型；首批类型为 `litellm`、`openai_compatible`、`anthropic`。
- [x] 供应商是模型的计费归属，调用连接是执行通道，两者不能混成一个字段。
- [x] 同一个供应商模型经不同连接调用时，建立不同真实模型记录，避免路由、价格和运行状态互相污染。
- [x] LiteLLM 和直连调用共用虚拟模型策略、降级逻辑、额度拦截和统计口径。
- [x] TokenPilot 是路由配置的唯一事实来源，不在 LiteLLM 内维护第二套业务路由。
- [x] 供应商密钥由应用环境变量或密钥系统保存；后台只保存 `credential_ref` 等非敏感引用。
- [x] 不建设承载提示词正文的中心代理；完全不改应用调用代码的代理模式不在本次范围内。
- [x] 不保留旧数据库、旧字段、双读、双写、升级或降级兼容逻辑；开发环境可以清库重建。
- [x] Web 继续使用 shadcn 组件和既有设计体系，保持字段少、术语少、功能完整，并完整支持中文和英文。

## 五、统一执行流程

### 一次调用的状态机

- [x] 定义统一状态机：生成 `operation_id` → 获取上下文 → 路由 → 预占额度 → 发起尝试 → 解析用量 → 结算/释放 → 上报事件。
- [x] 每次 fallback 前重新检查目标兼容性和剩余额度，禁止对同一尝试重复扣费。
- [x] 超时、网络错误、限流、供应商错误、用户取消和内容策略错误分别定义是否允许 fallback。
- [x] 流式调用只有在流结束或取消后结算；缺少供应商 usage 时采用明确的可配置策略，不伪造精确值。
- [x] 最终失败也要释放未使用预占、记录每次尝试，并返回不泄露连接密钥的可诊断错误。
- [x] 请求取消能向供应商传播，并确保后台任务不会在取消后重复结算。
- [x] 幂等键覆盖重试、SDK 本地 spool 重放和 API 重放，保证用量事件及额度结算 exactly-once 或等价幂等。

### LiteLLM 执行方式

- [x] 保留现有 LiteLLM callback/connector 能力，但把包内通用的标识、spool、批量上报、心跳、额度和配置代码提取成可复用内核。
- [x] LiteLLM 回调根据已发布快照解析虚拟模型和真实模型，不在 LiteLLM YAML 中复制业务路由。
- [x] 明确 LiteLLM 模型名称与 `request_model` 的映射，仅在 `litellm` 连接驱动内解释该名称。
- [x] LiteLLM 每次实际供应商尝试只生成一个事件；与 SDK fallback 共享 `operation_id` 时不得重复统计。
- [x] Connector 心跳携带实例、版本、连接绑定和配置版本，后台能显示连接是否正常接收配置。
- [x] LiteLLM 配置下发失败保留上一份可用配置，并在后台展示失败原因与版本差异。

### 直连执行方式

- [x] 建立供应商适配器接口，统一非流式、流式、超时、取消、用量解析、错误归类和能力描述。
- [x] 适配器只从应用本地凭证注册表按 `credential_ref` 取客户端或密钥，禁止从运行时快照获取秘密。
- [x] 应用启动时注册所有可能被路由到的连接；缺失凭证时给出连接名称和修复提示，不输出凭证值。
- [x] 允许用户注册已有 SDK Client，以复用自定义代理、连接池、重试和企业网关设置。
- [x] 适配器只能发送当前目标允许的请求参数；供应商专属参数通过严格命名空间和能力校验处理。
