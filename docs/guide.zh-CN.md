# 项目详细介绍

[English](guide.md)

## TokenPilot 解决什么问题

一个 AI 应用通常从一个模型名称和一把服务商密钥开始。业务增长以后，团队需要稳定回答一些单个服务商后台难以回答的问题：

- Token 是哪个应用、功能和用户消耗的？
- 主模型失败后改用备用模型，每一次尝试分别产生了多少花费？
- 服务商和模型版本变化时，业务代码能不能一直使用同一个模型名称？
- 不同模型的输入、输出、缓存和语音用量，怎样换算成统一的 AI Unit？
- 用户还剩多少 AI Unit，并发调用时怎样保证额度不会被重复使用？

TokenPilot 把这些规则集中到一个地方管理。应用可以通过 TokenPilot 的 Node 或 Python SDK 请求模型，也可以继续使用 LiteLLM，或者只上报其他客户端产生的调用。SDK 和 LiteLLM Connector 会发送不含提示词与回复内容的尝试记录；网络短暂中断时，记录先保存在本地可靠队列中。已发布的分流配置也会保留在本地，因此控制服务暂时不可用时，最后一次成功配置仍能继续工作。

## 主要功能区

### 应用与用户

每个应用都有独立的首页、用户、字段、接入密钥、报表、额度和发布状态。调用必须提供 `user_id`，推荐同时提供更易阅读的 `display_user`。新用户可以随调用自动出现，也可以在管理页面手工添加。类型明确的自定义字段可记录 `next_action`、`parse_context`、语音模式等业务信息，无须上传提示词和模型回复。

### 调用连接与真实模型

**调用连接**告诉可信运行程序怎样访问模型服务，其中包含调用方式、地址、能力，以及保存密钥的环境变量名称；TokenPilot 不保存密钥原文。**真实模型**是这个连接上可以实际调用的服务商模型，它的请求名称、模型花费备用规则和 AI Unit 换算率在一起维护。

### 虚拟模型与分流

**虚拟模型**是业务代码长期使用的稳定名称，例如 `fast-text` 或 `deep-reasoning`。虚拟模型中的候选项指向真实模型，并可设置默认顺序、失败后改用、时间条件、用户或用户组条件和临时切换。发布会生成不可修改的配置版本；可信运行程序下载并校验配置后，确认自己实际应用的版本。

因此，应用可以一直请求同一个虚拟模型，后台通过发布配置无感切换真实服务商或模型。请求包含图片或语音时，只会选择声明了相应能力的真实模型。

### 数据统计与报表

管理页面可以查看调用、完整尝试链、Token 明细、延迟、错误、模型花费、AI Unit 用量和配置覆盖缺口。筛选条件可以组合时间、应用、用户、用户标签、虚拟模型、真实模型、服务商、自定义字段和结果。保存后的分析可以加入每个应用自己的首页。统计始终读取 ClickHouse；PostgreSQL 保存配置和运行状态。

### 模型花费、AI Unit 与用户额度

模型花费表示模型服务实际产生的费用。SDK 或 Connector 上报的本次实际金额优先生效；调用端不能提供金额时，可以按内置字段或自定义属性设置有顺序的备用规则。AI Unit 是产品自己定义、用于统计和分配模型用量的独立单位，输入、输出、缓存、推理、图片、语音和自定义用量的换算率不受成本规则影响。缺少成本或 AI Unit 覆盖时会明确显示“未定价”或“未计算”，绝不会悄悄按零处理。

每个应用用户可以获得独立的 AI Unit 额度。强制限制会在调用前预留一个保守值，完成后按实际换算结果结算；没有发出模型请求时会释放预留。拉闸和重置额度都是明确且有审计记录的操作，不会删除历史。

## 隐私与可靠性边界

- 消息内容只在发起模型请求的程序内使用，上报不包含提示词和回复正文。
- 服务商密钥保存在运行程序的环境变量或团队已有的密钥系统中。
- SDK 和 Connector 使用本地 SQLite 可靠队列，通过稳定标识安全重试上报。
- PostgreSQL 与 ClickHouse 都是必需组件，分别承担配置和统计职责。
- 被拒绝的发布不会覆盖最后一次成功应用的配置。

## 推荐接入顺序

1. 部署 TokenPilot，完成首次设置。
2. 创建应用和应用接入密钥。
3. 定义允许应用上报的事件字段和用户字段。
4. 添加调用连接和真实模型，按需配置模型花费备用规则，并发布 AI Unit 换算率。
5. 创建虚拟模型，用代表性条件模拟结果后发布。
6. 接入 Node SDK、Python SDK或 LiteLLM Connector，并验证一次真实调用。
7. 检查用户、尝试链、模型花费、AI Unit 和保存的报表。
8. 先以只统计方式分配用户额度，确认身份与单价覆盖完整后，再启用提醒或强制限制。

这个顺序能让首次接入保持可观察、易纠正，同时不会把缺失数据伪装成正确结果。
