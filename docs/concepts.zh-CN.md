# 概念与计算原理

[English](concepts.md)

## 用量事件

Connector 会为每次模型尝试发送一个事件。事件包含稳定标识、模型身份、Token 和请求用量、耗时、结果、分流原因以及可信的产品上下文，不需要提示词或模型回复。Registry 对同一个事件标识只接收一次；重试会返回原结果，不会重复计算。

常见用量类型包括：

- `request`：一次模型尝试；
- `uncached_input_tokens`：没有命中缓存的输入 Token；
- `cache_read_tokens`：从服务商缓存读取的输入 Token；
- `cache_write_tokens`：写入服务商缓存的输入 Token；
- `reasoning_tokens`：支持该能力的服务商上报的推理 Token；
- `output_tokens`：模型可见输出 Token。

## 模型

TokenPilot 中的模型就是 LiteLLM 使用的模型名称，例如 `openai/gpt-4.1`。模型花费和 AIU 换算率都直接绑定这个真实模型，不再维护另一套基础模型或运行模型身份。

## 服务商成本

每个已发布成本配置会按用量类型定义单价和计价单位。计算只使用整数数量和十进制定点数：

```text
单项成本 = 用量数量 × 单价 / 单价单位
一次尝试成本 = 所有已匹配单项成本之和
```

所有非零用量都必须有价格。只要缺少一项，整次尝试就标记为未定价，避免部分成本看起来像完整成本。正式结果可以通过差额记录替换临时结果，历史记录保持不可变。

## AI Unit

AI Unit 使用独立单价表。一个简单示例可以是：

```text
1,000,000 输入 Token        = 1.000000 AIU
1,000,000 缓存读取 Token    = 0.250000 AIU
1,000,000 缓存写入 Token    = 0.500000 AIU
1,000,000 推理 Token        = 4.000000 AIU
1,000,000 输出 Token        = 3.000000 AIU
```

内部使用整数微 AIU 保存结果，只在规定边界进行一次舍入，不使用浮点数。每次计算都会保存实际使用的已发布单价快照、舍入方式和尝试计算规则。

服务商成本和 AI Unit 相互独立。一个模型可以已经配置成本但还没有 AI Unit，也可以已经有 AI Unit 但尚未录入服务商价格。覆盖页面会分别展示两类缺口。

## 虚拟模型与分流

虚拟模型是业务代码使用的稳定模型名称。它内部维护完整的分流设置，包括：

- 一个默认候选；
- 按顺序排列的失败候选；
- 零个或多个时间条件；
- 带明确过期时间的临时规则；
- 单调递增的策略修订号。

SDK 使用 ETag 拉取策略，并确认自己实际应用的精确修订。短暂故障时可以读取原子保存的最后可用策略。格式错误或已经过期的策略不会被静默接受。

## 额度生命周期

“只统计”模式只记录 AI Unit，不阻止请求。强制限制模式使用下面的过程：

```text
检查 → 预留 → 调用模型 → 结算
                    └──→ 释放
过期预留 ─────────────→ 过期
```

每个状态转换都可以安全重试。权威余额和变动记录保存在 PostgreSQL。ClickHouse 只接收用于统计的投影，不参与是否允许请求的决定。

## 数据由谁负责

| 数据                                | 权威位置                       |
| ----------------------------------- | ------------------------------ |
| 模型目录、价格、AI Unit、策略、密钥 | PostgreSQL                     |
| 计算结果、用户额度、预留、变动记录  | PostgreSQL                     |
| 队列租约和短期协调状态              | Redis                          |
| 报表和仪表盘                        | ClickHouse                     |
| Connector 重试缓冲                  | LiteLLM 旁边的本地 SQLite 文件 |

统计不会在 ClickHouse 故障时改读 PostgreSQL。明确显示统计不可用，比返回口径不同或不完整的数字更安全。

TokenPilot 不代理模型流量。LiteLLM 仍在请求链路中，TokenPilot 只接收用量事件，并发布分流、成本和 AIU 配置。

## 全新数据库规则

项目仍处于开发阶段。PostgreSQL 和 ClickHouse 都从空卷创建当前结构。发现未知对象或混合结构时会拒绝启动；开发环境应删除对应隔离卷，再创建当前结构。项目不保留旧结构转换逻辑。
