# 第一个应用完整教程

[English](tutorial.md)

本教程从空白管理页面开始，完成一次真实 LiteLLM 调用、不含内容的用量上报、模型花费与 AIU 统计、用户额度和分流发布。Mac 示例直接运行 Python，不需要本机容器环境。

## 1. 创建应用和密钥

完成首次配置后，新建一个应用，只需要填写名称，URL 标识由 TokenPilot 自动生成。在**设置 → 接入密钥**中分别创建三把应用密钥：

- 用量上报和 Connector 心跳；
- 运行配置和 AIU 预留；
- 用于验证的统计只读密钥。

密钥原文只显示一次，请在关闭对话框前保存好。

## 2. 定义字段

打开**字段**，添加下面的示例：

| 范围 | 字段            | 类型  |
| ---- | --------------- | ----- |
| 事件 | `next_action`   | 文本  |
| 事件 | `voice_enabled` | 是/否 |
| 用户 | `parse_context` | 文本  |
| 用户 | `voice_type`    | 文本  |

敏感字段应标记为敏感。未定义字段或类型错误的值会被拒绝，这样搜索和报表的含义才稳定。

## 3. 录入模型和换算率

打开**模型**。新增模型只需要显示名称和准确的 LiteLLM 名称。随仓库提供的原生示例使用：

- `openai/local-success`；
- `openai/local-primary`；
- `openai/local-fallback`。

进入每个模型，填写服务商成本和 AIU 换算率。AIU 直接绑定这个模型；输入、输出、缓存、推理、图片、语音等支持的用量可以设置不同换算率。

## 4. 创建并发布虚拟模型

打开**虚拟模型**，创建业务代码使用的稳定名称，先添加主模型，再添加备用模型。可选条件可以按时间、用户、用户组或临时切换选择候选模型。使用**模拟**输入一个代表性用户，确认结果后在**发布**页面下发。

只有目标 Connector 确认应用了完全相同的配置版本，页面才显示已生效。拒绝的配置会保留拒绝原因。

## 5. 运行 Mac 原生 LiteLLM 示例

在仓库根目录执行：

```bash
cd examples/litellm-local
cp .env.example .env
# 填写三把应用密钥和应用标识。
uv sync --all-groups
uv run python app.py
uv run python verify_reporting.py
```

示例会启动兼容 OpenAI 的本地假服务，使用真实 LiteLLM Router，完成一次成功调用和一次主模型失败后改用备用模型的调用，再通过带持久缓冲的 Connector 上报。`.env.example` 已包含可选的局域网代理写法。

## 6. 验证用户和统计

打开**用户**。上报中的 `user_id` 已经自动出现在当前应用，推荐填写的 `display_user` 会作为名称显示。后台也可以只填写 `user_id` 手工新增用户；用户始终只属于当前应用。

继续检查**用量明细**、**模型花费**和 **AIU**，按示例用户和模型筛选。详情应展示 Token、实际备用模型、分流原因、成本、AIU 和类型化字段，但不能出现提示词或模型回复。可以保存当前分析并加入当前应用首页。

## 7. 验证额度控制

为示例用户设置 AIU 额度。严格模式会在调用前预留，成功后按实际用量结算，失败时释放。重置额度会追加带审计的调整记录，不删除历史。拉闸状态会进入运行快照，后续调用被阻止，同时 Connector 在短暂故障时仍保留最后一次成功应用的配置。
